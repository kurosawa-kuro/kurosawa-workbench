# 01 要件 — 黒澤 Workbench

## サイトの位置づけ

**企業ホームページの AI QA を、黒澤俊文個人の営業窓口に移植したもの。**

一般企業のサイトでは「このサービスで対応できますか？」「この課題は相談対象ですか？」
「問い合わせ前に整理してください」という AI QA が普通に置かれている。
黒澤 Workbench はそれを、フリーランス技術コンサルタント個人の案件受付・初回ヒアリング補助として実装する。

```
一般企業サイト:    自社サービス説明 + FAQ + 問い合わせ誘導
黒澤 Workbench:   黒澤の対応領域説明 + 案件相談 QA + 問い合わせ文生成
```

## 訪問者が感じる障壁（AI QA が解消するもの）

- この人に相談していい案件なのか分からない
- 技術的に合っているのか分からない
- どこまで任せられるのか分からない
- 問い合わせ文を書くのが面倒

## 黒澤のプロフィール（AI QA の根拠）

- **専門**: GCP AI 基盤（Vertex AI Pipelines / Vertex Vector Search / BigQuery）
- **強み**: 上流コンサル（ヒアリング・要件定義・PoC・スライド）から実装・デプロイまで一気通貫
- **稼働**: フルリモート専門（札幌在住）/ 月 90〜130 万円
- **実績**: 大手製造業向け GCP MLOps 基盤整備チームリーダー（2026〜現在）

## 優先順位

```
AI 案件相談 QA の動作（ルール判定 + LLM 判定 + 問い合わせ文生成）
↓ 問い合わせ導線（draftInquiry コピー → mailto: 送信）
↓ 黒澤プロファイルの正確な反映
↓ AI 障害時フォールバック
↓ 実績ポートフォリオ・Contact ページ（後続）
```

## ユースケース

| ID | ユースケース | 成功条件 |
|---|---|---|
| UC-001 | GCP AI 基盤 / MLOps 案件の対応可否を確認する | fit: high / summary に Vertex AI Pipelines 等の具体名が含まれる |
| UC-002 | 上流コンサル（要件定義・PoC・スライド）だけ依頼できるか確認する | canHandle: true / suggestedScope に「ヒアリング」「PoC 設計」が含まれる |
| UC-003 | 常駐・出社必須の案件を入力する | fit: ng / LLM を呼ばずに即返答 |
| UC-004 | 対応外案件（DL 研究専業・デザイン専業等）を入力する | fit: ng / 理由が明示される |
| UC-005 | 推奨問い合わせ文をコピーして送る | draftInquiry が1クリックでコピーでき、mailto: リンクで送れる |
| UC-006 | AI 障害時も詰まらない | フォールバックメッセージと問い合わせ先が表示される |

## 入力フィールド

| フィールド | 必須 | 説明 |
|---|---|---|
| 相談内容（案件概要） | 必須 | 作りたいもの・課題・技術スタック等を自然文で |
| 技術スタック | 任意 | 使いたい / 使っているスタック |
| 予算・単価感 | 任意 | 目安または未定 |
| 期限・稼働期間 | 任意 | 希望納期・参画期間 |
| 既存コードの有無 | 任意 | ゼロから / 既存あり |
| 希望する関わり方 | 任意 | 丸投げ / 上流のみ / 相談しながら / 一部支援 等 |

## AI レスポンス スキーマ

```json
{
  "fit": "high | medium | low | ng",
  "canHandle": true,
  "summary": "1〜2文で対応可否と理由",
  "suggestedScope": ["ヒアリング・要件整理", "Vertex AI Pipelines 設計", "Cloud Run サービング"],
  "risks": ["フルリモート専門のため出社不可", "◯◯はスコープ調整が必要"],
  "questions": ["GCP プロジェクトはすでにありますか？", "チームリーダーとしての参画ですか？"],
  "draftInquiry": "訪問者がそのままコピーして送れる問い合わせ文"
}
```

## NG 判定ルール（LLM を呼ぶ前にルールで弾く）

以下の条件が入力に含まれる場合、LLM を呼ばず即座に `fit: ng` を返す。

| NG 条件 | 判定理由 |
|---|---|
| 常駐・出社必須・週 ◯ 出社 | フルリモートのみ対応 |
| デザイン専業・UI/UX デザイン | 対象外 |
| DL モデル研究・論文実装専業 | 対象外（周辺インフラは対応可） |
| 薬機法対応の医療・美容効能表現 | 対象外 |
| 実決済・個人情報の本番管理専業 | 対象外（PoC・設計レビューは対応可） |

## 問い合わせ導線（最終ゴール）

AI QA の目的は会話ではなく**整理済みリード獲得**。最後は必ず以下へ誘導する。

1. `draftInquiry` をコピー
2. `mailto:` リンクでメール送信（Contact バックエンドなし、MVP はこれで十分）
3. X / GitHub / Wantedly 等の外部プロフィールへ遷移（任意）

## 制約

| 区分 | 制約 |
|---|---|
| セキュリティ | API キーはブラウザに露出しない。Edge Functions 経由でのみ呼び出す |
| コスト | 入力文字数制限を設ける。NG 判定はルールで先行し LLM 呼び出しを減らす |
| 耐障害性 | AI 障害時は「現在 AI が応答できません。直接お問い合わせください（mailto:）」を表示 |
| データ | 案件情報はサーバーサイドに保存しない。AI API に渡すのみ |
