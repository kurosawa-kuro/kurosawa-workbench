# 店舗側デモ画面 /admin-demo

## Goal

`/admin-demo` に「AI 経由注文数・よくある相談・売れ筋・AI 改善提案」を表示する。

## Value

単なる EC デモではなく「店舗改善 AI デモ」に見える。営業での差別化ポイント。

## Scope

- セッション内の localStorage データから集計して表示（DB なし）
- 今日の相談件数・AI 経由注文数
- よくある相談トップ 3（入力チップの使用回数など）
- 売れ筋商品（カート追加・注文完了データから）
- AI 改善提案（固定文言でも可）

## Non-scope

- リアルタイム集計・WebSocket
- 認証・管理者ログイン

## Done

- セッション中に注文やコンシェルジュ相談をした後、admin-demo に反映される
- 固定文言の AI 改善提案が表示される

## Evidence

目視確認

## Stop / Ask Owner If

- 特になし。固定文言で十分であれば実装量は少ない
