# どのリポジトリを参考にするか

## Goal

手元の既存 EC リポジトリからコピー流用できるコードを特定し、実装コストを下げる。

## 調査済みリポジトリ一覧

| リポジトリ | 場所 | 技術スタック | EC 関連コード |
|---|---|---|---|
| data-driven-fargate-ecshop | `ec-shop-sample/sample/data-driven-fargate-ecshop/fullstack-nextjs/` | Next.js + Tailwind + DB + 認証 | ProductCard / cart / checkout / chatbot / search / products / admin |
| eks-app-mlops ec-shop | `eks-app-mlops-platform-v2.0/apps/ec-shop/frontend/` | Next.js + Tailwind + DB + 認証 | ProductCard / cart / orders / products / LLM components |
| admin-pico | `ec-shop-sample/sample/admin-pico/` | Pico.css + 静的 HTML（React なし） | analytics / orders / customers ダッシュボード UI |
| data-driven-cloud-ecshop | `ec-shop-sample/sample/data-driven-cloud-ecshop/` | Next.js + Tailwind + DB | EC バックエンド中心 |

## 結論

**流用価値がある：2 箇所**

---

### 1. ProductCard + カート・チェックアウト UI ロジック

**出典**: `data-driven-fargate-ecshop` または `eks-app-mlops ec-shop`

**流用できるもの**:
- ProductCard のデータ構造（`id` / `name` / `price` / `imageUrl` / `rating`）
- 商品一覧・詳細の表示パターン
- カート → 注文確認 → 注文完了の UX フロー（localStorage に置き換え）
- chatbot UI の会話コンポーネント構造 → AIコンシェルジュに転用

**注意点（今回の差分）**:
- Next.js → React + Vite への書き換えが必要（`Link`, `Image` など）
- Tailwind クラス → Pico.css + `style.css` に置き換え
- DB / API 呼び出し → 固定 JSON + localStorage に差し替え
- 認証ガード（`(protected)` レイアウト）は全部削除

---

### 2. admin-demo 画面の UI パターン

**出典**: `ec-shop-sample/sample/admin-pico/src/pages/admin/`

**流用できるもの**:
- `analytics.html` → 相談件数・AI経由注文・売れ筋のカード/グラフ UI
- `orders.html` → よくある相談・注文一覧テーブル UI
- Pico.css を使った同一 CSS フレームワーク → 今回と完全一致

**注意点**:
- 静的 HTML → React JSX に書き直しが必要
- データはダミー固定 or localStorage から読む形に変える

---

## 流用しないもの

| リポジトリ | 理由 |
|---|---|
| data-driven-cloud-ecshop | バックエンド中心。フロントは薄い |
| k8s-ec-llm-analytics | K8s / インフラ寄り。今回不要 |
| eks-app-mlops ec-shop の LLM 層 | 複雑すぎる。Supabase Edge Function で十分 |

## 作業方針

1. **先に products.js を作る**（タスク 02）。データ構造を決めてから既存コードを参照すると合わせやすい。
2. **ProductCard は fargate-ecshop を原型に**、Tailwind → Pico.css + style.css に書き直す。
3. **chatbot UI は fargate-ecshop の `(public)/chatbot` を参照**して AIコンシェルジュ（タスク 04）に転用する。
4. **admin-demo は admin-pico の `analytics.html` + `orders.html` を参照**して JSX に起こす（タスク 08）。
