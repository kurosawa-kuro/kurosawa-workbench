# Bounded Change Prompt

Inspect whether the catalog remains a present-state mirror rather than a plan.

Rules:
- Keep Evidence read-only.
- Keep machine join keys such as `evidence_ids` out of the Markdown body.
- Keep uncertainty visible as descriptive current implications instead of inventing certainty.
- Do not include rollback, failure conditions, next actions, or recommendations.
- Do not generate target-repo tasks, checklists, branches, or change artifacts.
- Do not include secret values.
